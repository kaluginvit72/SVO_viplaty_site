# Опрос «Расчёт» (предварительный расчёт выплат)

Кнопка на сайте: **«Узнать, какие выплаты положены»**. На экране прогресса: бейдж **«Ориентир по выплатам»**.

Источник текстов: `web/src/data/texts/quiz-copy.ts` (`stepQuestions`, варианты ответов, `quizFlowCopy`).

## Порядок шагов

Цепочка задаётся в `web/src/lib/quiz/steps.ts` для режима **`fresh`**:

1. `deceased`  
2. `relation`  
3. **`relation_complex`** — только если на шаге 2 выбрано «Ситуация нестандартная или спорная» (`complex`)  
4. `recipients`  
5. **`recipients_count`** — только если на шаге 4 выбрано «Четверо и больше» (`me_plus_4_or_more`)  
6. `documents` (можно отметить **несколько** вариантов, затем **«Далее»**)  
7. `region` (свободный ввод в поле)

**Число шагов на экране:** от **5** (без веток 3 и 5) до **7** (обе ветки включены).

## 6. Документы

**Идентификатор шага:** `documents`

**Вопрос:** Что из документов уже есть на руках? Можно отметить несколько вариантов.

**Тип ввода:** мультивыбор (чекбоксы), затем кнопка **«Далее»**.

**Варианты ответов** (можно выбрать несколько):

| Значение (id) | Текст на экране |
|---------------|-----------------|
| `death_cert` | Свидетельство о смерти |
| `unit_notice` | Извещение или документы из части |
| `kinship_docs` | Документы, подтверждающие родство |
| `almost_all` | Почти всё уже собрали |
| `nothing_yet` | Пока ничего не собирали |
| `unsure` | Пока не понимаю, что именно нужно |

**Группа для доступности (aria):** «Какие документы уже есть».

---

## 7. Регион

**Идентификатор шага:** `region`

**Вопрос (заголовок экрана):** В каком регионе вы живёте?

**Поле ввода:**

| Элемент | Текст |
|---------|--------|
| Подпись поля (`regionLabel`) | Регион проживания |
| Плейсхолдер (`regionPlaceholder`) | Например: Краснодарский край |
| Подсказка под полем (`regionHint`) | Достаточно области, края или республики — так точнее ориентир по местным мерам поддержки. |

**Тип ввода:** одна строка текста (не фиксированный список вариантов).

---

## Кнопки и подсказки интерфейса (фрагмент `quizFlowCopy`)

- **«Далее»** — после шага с документами и при необходимости на других шагах с явным переходом.  
- **«Показать предварительный расчёт»** — финальный переход к результату.  
- **«Назад»** — возврат на предыдущий шаг.  
- **«Очистить расчёт»** — сброс с подтверждением (`resetConfirm`).  
- Прогресс: «Шаг *n* из *total*» (`progressQuestion`).

---

## Примечание для разработки

В `quiz-copy.ts` также заданы массивы **`statusNowOptions`**, **`submittedWhereOptions`**, **`waitingOptions`** — для старых/смежных сценариев в данных и резолверах подписей; в **текущем** пошаговом UI опроса «Расчёт» в `QuizFlow.tsx` эти шаги **не показываются**. Если снова включат отдельную ветку «застряли» в том же виде, варианты уже описаны в коде.
